|------------|------------|

## Быстрый старт

### Первоначальная настройка (один раз)

```powershell
cd D:\YandexDisk\Coding\AISigma

# Установка Git hooks
.\scripts\setup-hooks.ps1
```

### Ежедневный workflow

```powershell
# 1. Начало работы — запуск серверов
.\scripts\dev-start.ps1

# 2. После изменений — что нужно сделать?
.\scripts\dev-check.ps1

# 3. Деплой — полный пайплайн с подтверждениями
.\scripts\dev-deploy.ps1

# Или быстрый push (если всё проверено)
.\scripts\quick-push.ps1 -m "feat: описание изменений"
```

## Скрипты

### dev-start.ps1

Запускает backend (Flask) и frontend (Vite) в двух отдельных окнах PowerShell.

**Использование:**
```powershell
.\scripts\dev-start.ps1
```

**Результат:**
- Окно 1: Flask на http://localhost:5000
- Окно 2: Vite на http://localhost:5173
- При первом запуске при необходимости выполняется `npm install` в frontend

---

### dev-check.ps1

Анализирует изменённые файлы и показывает, какие действия требуются.

**Использование:**
```powershell
.\scripts\dev-check.ps1
```

**Пример вывода:**
```
=== AISigma Dev Check ===

Изменённые файлы:
  app/config/system_prompt.txt
  frontend/src/styles/chat.css

[FRONTEND] Изменены файлы фронтенда
  -> Перед деплоем: npm run build в frontend/
[BACKEND]  Изменены файлы бэкенда
  -> Перезапустите Flask (Ctrl+C, затем python -m app.main)

[!] ВНИМАНИЕ: frontend/ изменён, но app/static/ не обновлён.
```

---

### dev-deploy.ps1

Интерактивный пайплайн деплоя. Запрашивает подтверждение на каждом шаге.

**Использование:**
```powershell
.\scripts\dev-deploy.ps1

# С предзаданным сообщением коммита
.\scripts\dev-deploy.ps1 -CommitMessage "feat: новая функция"
```

**Шаги:**
1. Анализ изменений
2. Сборка фронтенда (если изменён) — подтверждение [y/N]
3. Просмотр diff — подтверждение [y/N]
4. Ввод сообщения коммита (если не передано)
5. Push — подтверждение [y/N]

---

### quick-push.ps1

Быстрый коммит и push. Для случаев, когда изменения уже проверены локально.

**Использование:**
```powershell
.\scripts\quick-push.ps1 -m "fix: исправить обработку модели X"
```

**Особенности:**
- Если `frontend/src/` изменён, но `app/static/` не обновлён — выдаёт предупреждение и спрашивает подтверждение
- Добавляет все изменения, создаёт коммит, пушит

---

### setup-hooks.ps1

Копирует pre-commit и pre-push хуки из `scripts/` в `.git/hooks/`.

**Использование:**
```powershell
.\scripts\setup-hooks.ps1
```

Запустите один раз после клонирования репозитория или при первом использовании скриптов.

---

## Git hooks

### pre-commit

Срабатывает перед каждым `git commit`.

**Проверки:**
- `.env` не должен попадать в коммит (блокирует коммит)
- Синтаксис Python-файлов в `app/` (блокирует при ошибках)

### pre-push

Срабатывает перед каждым `git push`.

**Проверка:**
- Если `frontend/src/` изменён в коммитах, но `app/static/` не обновлён — выдаёт предупреждение
- Пользователь может продолжить (y) или отменить (N)

## Сценарии использования

### Сценарий 1: Правка промпта и стилей

**Ситуация:** Изменены `app/config/system_prompt.txt` и `frontend/src/styles/chat.css`.

1. Запустить `.\scripts\dev-start.ps1` (если ещё не запущено)
2. Агент вносит изменения и уведомляет о необходимости перезапуска Flask
3. Перезапустить Flask в его окне (Ctrl+C → `python -m app.main`)
4. Стили обновятся автоматически (Vite HMR)
5. Проверить в браузере
6. Запустить `.\scripts\dev-deploy.ps1` → подтвердить сборку → коммит → push

---

### Сценарий 2: Быстрый фикс только в бэкенде

**Ситуация:** Исправлен баг в `app/api/routes.py`.

1. Перезапустить Flask
2. Проверить локально
3. `.\scripts\quick-push.ps1 -m "fix: описание бага"`

Сборка не требуется — push занимает ~10 секунд.

---

### Сценарий 3: Только правка промптов

**Ситуация:** Изменён `app/config/style_ia_prompt.txt`.

1. Перезапустить Flask
2. `.\scripts\quick-push.ps1 -m "chore: обновить стилевые инструкции"`

Фронтенд не затронут — предупреждений не будет.

## Рекомендации по Git

- **`git config pull.rebase true`** — линейная история при pull
- **Атомарные коммиты** — один коммит = одно логическое изменение
- **Частые пуши** — меньше конфликтов, проще откат
- **`git stash`** — при переключении контекста сохранять незавершённую работу
